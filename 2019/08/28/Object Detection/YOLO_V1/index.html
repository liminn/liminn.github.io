YOLO_V1

发表于 2019-08-28 | 分类于 Object Detection

摘要

  • 提出YOLO，将目标检测任务定义成一个回归问题，直接从图像回归出包围边框坐标和类别概率。通过YOLO，你只需看一次(YOLO, you only look once)图像，就可预测出图像中有什么物体，以及物体在哪儿。
  • YOLO统一的模型结构，使其有如下优点：
    • YOLO极快，能够实时进行目标检测。
    • YOLO在训练和测试阶段，都能编码整幅图像的上下文信息；因此，相比于Fast R-CNN，YOLO会出现更少的背景错误。
  • YOLO的缺点是：
    • 相比于state-of-the-art的检测系统，其效果仍有差距。
    • 对于小物体，检测效果很差。

关键点

YOLO整体架构


如上图所示，为YOLO整体结构：

  • 整体来看，Yolo算法采用一个单独的CNN模型实现end-to-end的目标检测，整个系统如图5所示：首先将输入图片resize到448x448，然后送入CNN网络，最后处理网络预测结果得到检测的目标。

  • 首先，YOLO将输入图片分成$S \times S$个grid。如果目标的中心落在某个grid cell里面，那么这个grid cell就为检测这个目标负责。
  • 一方面，每一个grid cell预测B个bounding box，每个bounding box预测五个值: $x,y,w,h$以及confidence score：
    • 其中，confidence score一方面反映出模型有多确定这个bounding box里面包含物体，另一方面反映出模型认为预测出的这个bounding box的位置有多准确。将confidence score定义为：$\text{confidence} = P(\text{Object})\times \text{IOU}_{\text{pred}}^{\text{truth}}$。如果某个grid cell中不存在物体，那么confidence score应该为0。否则，confidence score应该为预测出的bounding box和GT box之间的IOU值。
    • 其中，(x,y)为为预测的bounding box的中心坐标。(x,y)是相对于grid cell左上角坐标点的偏移值，并且偏移值的度量单位是相对于grid cell大小而言，因此(x,y)的取值范围在[0,1]。
    • 其中，(w,h)为预测的bounding box的宽高。(w,h)的值是相对于整个图片宽高的比例，因此(w,h)的取值范围在[0,1]。
  • 另一方面，每一个grid cell预测C个条件类别概率$P(\text{Class}_i|\text{Object})$，即条件为这个grid cell包含物体。注意，与B的数值无关，是对每个grid cell产生一组类别概率。
  • 在推理阶段，将grid cell预测的条件类别概率$P(\text{Class}_i|\text{Object})$和预测的每个bounding box的confidence score单独相乘：
    $\text{confidence}\times P(\text{Class}_i|\text{Object}) = P(\text{Object})\times \text{IOU}_{\text{pred}}^{\text{truth}}\times P(\text{Class}_i|\text{Object})= P(\text{Class}_i)\times \text{IoU}_{\text{pred}}^{\text{truth}}$,
    即得到每个bounding box的class-specific confidence score。该score编码了各类别物体出现在bounding box中的概率，以及bounding box与object的位置匹配程度。
  • 因此，YOLO最终预测出的tensor维度为：$S\times S \times (B\times 5+C)$。
    • YOLO在PASCAL VOC进行评估时，使用$S=7$, $B=2$。由于PASCAL VOC共有20类，故$C=20$。因此，YOLO最终预测出的tensor维度为$7×7×30$。

如上图所示，为YOLO的网络结构：

  • Yolo采用卷积网络来提取特征，然后使用全连接层来得到预测值。网络结构参考GooLeNet模型，包含24个卷积层和2个全连接层，如图8所示。对于卷积层，主要使用1x1卷积来做channle reduction，然后紧跟3x3卷积。对于卷积层和全连接层，采用Leaky ReLU激活函数： 。但是最后一层却采用线性激活函数。

损失函数


如上图所示，为YOLO的损失函数：

  • 其中，$\mathbb{1}_{i}^{obj}$表示：第i个grid cell中存在物体；$\mathbb{1}_{ij}^{obj}$表示：第i个grid cell 中的第j个预测的bounding box为预测该grid cell中的物体负责。

YOLO的损失函数简述为：

  • 对于存在物体的grid cell，对为预测该物体负责的bounding box，即$\mathbb{1}_{ij}^{obj}$:
    • 计算其中心坐标的平方和误差(sum-squared error)。
    • 计算其宽高的平方和误差
    • 计算其置信度的平方和误差
  • 对于不存在物体的grid cell，对其所有预测bounding box，即$\mathbb{1}_{ij}^{noobj}$：
    • 计算其置信度的平方和误差
  • 对于存在物体的grid cell，即$\mathbb{1}_{i}^{obj}$：
    • 计算其各类别概率的平方和误差
  • 注意：
    • 对于定位误差(localization error)和分类误差(classification error)：直接采用均方和误差，会平等地对待两项误差，采用较大的权重$\lambda_{\text{coord}} = 5$来更强调定位误差。
    • 对于“不包含目标的grid cell中预测bounding box的置信度”和“包含目标的grid cell中预测bounding box的置信度”：由于前置数量远多于后置，因此对于前者，采用较小的权重值$\lambda_{\text{noobj}} = 0.5$来减小其损失值。否则，会导致模型不稳定，在训练早期就发散。
    • 对于大的bounding box和小的bouding box：直接采用均方和误差，会同等对待大小不同的bounding box，但是实际上较小的bounding box的偏差值应该要比较大的bounding box的偏差值要更敏感。为了保证这一点，将预测bounding box的宽与高$(w,h)$，改为预测对宽与高的平方根值$(\sqrt{w},\sqrt{h})$。
    • YOLO对每个grid cell会预测多个bounding box。在训练阶段，只希望其中的一个bounding box来为预测该物体负责。因此，文中选用与GT box的IOU最大的那个预测bounding box，来为预测该物体负责。这样设置将会每一负责的预测bounding box更加专业化，其可以分别适用不同大小，不同高宽比的目标，从而提升模型性能。
    • 疑问：大家可能会想如果一个单元格内存在多个目标怎么办，其实这时候Yolo算法就只能选择其中一个来训练，这也是Yolo算法的缺点之一。
  • 可以注意到：损失函数只会对存在物体的grid cell，惩罚分类误差；只会对预测该物体负责的预测bounding box(与GT box有最高的IOU)，惩罚坐标误差。

训练流程


如上图所示，为YOLO的网络流程：

  • 在训练之前，先在ImageNet上进行了预训练，其预训练的分类模型采用图8中前20个卷积层，然后添加一个average-pool层和全连接层。
  • 预训练之后，在预训练得到的20层卷积层之上加上随机初始化的4个卷积层和2个全连接层。由于检测任务一般需要更高清的图片，所以将网络的输入从224x224增加到了448x448。

在PASCAL VOC 2007的训练集/验证集/测试集，以及PASCAL VOC 2012的训练集/验证集上进行训练，在PASCAL VOC 2012的测试集上进行测试：

  • 训练135个epoch, batch size为64, momentum为0.9, weight_deacy为0.0005
  • 学习率策略：
    • 在第一个epoch，缓慢地将学习率从$10^{-3}$提升到$10^{-2}$。如果起始就采用一个高学习率，那模型通常会由于不稳定的梯度而发散。
    • 采用学习率为$10^{-2}$训练75个epoch，采用学习率为$10^{-3}$训练30个epoch，采用学习率为$10^{-4}$训练最后30个epoch。
  • 为了避免过拟合，采用drop和广泛的数据扩充(extensive data augmentation)：
    • 在第一个全连接层后，施加0.5的dropout层
    • 随机缩放图像，最大为1.2
    • 在HSV颜色空间，随机调整图像的曝光(exposure)和饱和度(saturation)，最高为1.5

实验结果


如上图所示，在PASCAL VOC 2007数据集上，Yolo与其它检测算法做了对比，包括DPM，R-CNN，Fast R-CNN以及Faster R-CNN：

  • 与实时性检测方法DPM对比，可以看到Yolo算法可以在较高的mAP上达到较快的检测速度，其中Fast Yolo算法比快速DPM还快，而且mAP是远高于DPM。
  • 但是相比Faster R-CNN，Yolo的mAP稍低，但是速度更快。
  • 所以。Yolo算法算是在速度与准确度上做了折中。


如上图所示，为YOLO和Fast R-CNN的误差对比分析：

  • 可以看到，Yolo的Correct的是低于Fast R-CNN。另外Yolo的Localization误差偏高，即定位不是很准确。
  • 但是Yolo的Background误差很低，说明其对背景的误判率较低。
