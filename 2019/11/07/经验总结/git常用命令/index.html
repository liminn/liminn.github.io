git常用命令

发表于 2019-11-07 | 分类于 工具使用总结

xx

常用命令

  • git status: 查看仓库的状态

  • git init: 初始化Git仓库

  • git add xxx: 将xxx文件添加到Git仓库的临时缓冲区

  • git commit -m "text commit"命令，将xxx文件提交到Git仓库：

  • git log: 打印 Git 仓库提交日志

  • git branch: 查看Git仓库的分支情况

  • git branch -r: 查看远程主机的分支名

  • git branch x: 创建了一个名为x的分支，并且当前的位置仍然为主分支

  • git checkout x: 切换到x分支

  • git checkout -b x: 创建分支x的同时，直接切换到新分支x

  • git merge x: 切换到master分支，然后输入git merge x命令，将x分支合并到master分支：

  • git branch -d x: 删除x分支

  • git branch -D x: 有时，通过git branch -d x命令可能出现删除不了现象，例如分支a的代码没有合并到主分支等，通过命令git branch -D x进行强制删除

关于git config:

  • git config --global --list: 查看全局(当前用户)的配置信息
  • git config --local --list: 查看局部(当前仓库)的配置信息
  • git config --global user.name "myname": 配置局部(当前仓库)的用户名信息
  • git config --global user.email "test@gmail.com": 配置局部(当前仓库)的用户邮箱信息
  • git config --local user.name "名字": 配置全局(当前用户)的用户名信息
  • git config --local user.email "邮箱": 配置全局(当前用户)的用户邮箱信息

关于git push <远程主机名> <本地分支名>:<远程分支名>:

  • git push origin master: 如果远程分支被省略，如上则表示将本地分支推送到与之存在追踪关系的远程分支（通常两者同名），如果该远程分支不存在，则会被新建。因此该行命令是将本地的master分支推送到origin主机的master分支。如果master不存在，则会被新建。
  • git push origin : master: 如果省略本地分支名，则表示删除指定的远程分支，因为这等同于推送一个空的本地分支到远程分支。
  • git push origin --delete master: 与上行命令等同，删除origin主机的master分支。
  • git push origin: 如果当前分支与远程分支之间存在追踪关系，则本地分支和远程分支都可以省略。该行命令为将当前分支推送到origin主机的对应分支。
  • git push: 如果当前分支只有一个远程分支，那么远程主机名都可以省略。
  • git push -u origin master: 如果当前分支与多个主机存在追踪关系，则可以使用-u选项指定一个默认主机，这样后面就可以不加任何参数使用git push。不带任何参数的git push，默认只推送当前分支，这叫做simple方式。
  • git push --all origin: 不管是否存在对应的远程分支，将本地的所有分支都推送到远程主机，这时需要使用–all选项。
  • git push --force origin: 如果远程主机的版本比本地版本更新，推送时Git会报错，要求先在本地做git pull合并差异，然后再推送到远程主机。这时，如果你一定要推送，可以使用–force选项。
    上面命令使用-–force选项，结果导致在远程主机产生一个”非直进式”的合并(non-fast-forward merge)。除非你很确定要这样做，否则应该尽量避免使用–-force选项。
  • git push origin HEAD: 将当前分支推送到远程的同名的简单方法
  • git push origin HEAD:branchx: 将当前分支推送到远程主机的指定分支的简单方法。 这种形式方便推送当前分支，而不考虑其本地名称。如果远程主机的指定分支不存在，则新建该分支。
  • git push origin lbranch-1:rbranch-1: 推送本地分支lbranch-1到新的远程分支rbranch-1。如果远程主机的指定分支不存在，则新建该分支。
  • git push -f origin lbranch-1:rbranch-1: 用本地分支lbranch-1覆盖远程分支rbranch-1。
